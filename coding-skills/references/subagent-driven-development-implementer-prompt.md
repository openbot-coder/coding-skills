# 实现者子智能体提示模板

在派遣实现者子智能体时使用此模板。

```
Task 工具 (通用):
  description: "实现任务 N: [任务名称]"
  prompt: |
    你正在实现任务 N: [任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 粘贴在这里，不要让子智能体读文件]

    ## 背景

    [场景设定：此任务的位置、依赖关系、架构背景]

    ## 开始之前

    如果你对以下方面有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在就提出来。** 在开始工作前提出任何疑虑。

    ## 你的工作

    一旦你明确了需求：
    1. 严格按照任务规格实现
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实现可以工作
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 汇报

    工作目录: [目录]

    **工作期间：** 如果遇到意外或不清楚的情况，**提问**。
    暂停并澄清总是可以的。不要猜测或做假设。

    ## 汇报前：自我审查

    用全新的视角审查你的工作。问自己：

    **完整性：**
    - 我是否完全实现了规格中的所有内容？
    - 是否遗漏了任何需求？
    - 是否有未处理的边界情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（匹配事物的功能，而非实现方式）？
    - 代码是否干净且可维护？

    **纪律：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了被要求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而非仅仅是 mock 行为）？
    - 我是否按要求遵循了 TDD？
    - 测试是否全面？

    如果在自我审查中发现问题，在汇报前立即修复。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 测试了什么以及测试结果
    - 修改了哪些文件
    - 自我审查发现（如有）
    - 任何问题或疑虑
```
