# 规格合规审查提示模板

在派遣规格合规审查子智能体时使用此模板。

**目的：** 验证实现者构建了被要求的内容（不多不少）

```
Task 工具 (通用):
  description: "审查任务 N 的规格合规性"
  prompt: |
    你正在审查实现是否符合其规格。

    ## 需求内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得异常快速。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 轻信他们关于实现内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要做：**
    - 阅读他们实际写的代码
    - 逐行对比实际实现与需求
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没有提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被要求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些功能可用但实际上没有实现？

    **额外/不必要的工作：**
    - 他们是否构建了未被要求的内容？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规格中没有的"锦上添花"功能？

    **误解：**
    - 他们是否对需求有不同的理解？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式不对？

    **通过阅读代码验证，而非信任报告。**

    报告：
    - ✅ 符合规格（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附文件:行号引用]
```
