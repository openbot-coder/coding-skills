# 执行计划

## 概述

加载计划，批判性审查，分批执行任务，在批次之间报告审查。

**核心原则：** 批量执行 + 架构师审查检查点。

**开始时宣布：** "我正在使用执行计划技能来实现这个计划。"

## 流程

### 第 1 步：加载并审查计划
1. 读取计划文件
2. 批判性审查 - 识别关于计划的问题或顾虑
3. 如有顾虑：在开始前向人类搭档提出
4. 如无顾虑：创建 TodoWrite 并继续

### 第 2 步：执行批次
**默认：前 3 个任务**

对每个任务：
1. 标记为 in_progress
2. 严格按步骤执行（计划已拆分为小步骤）
3. 按指定运行验证
4. 标记为已完成

### 第 3 步：报告
批次完成时：
- 展示已实现的内容
- 展示验证输出
- 说："等待反馈。"

### 第 4 步：继续
根据反馈：
- 如需要则应用修改
- 执行下一批次
- 重复直到完成

### 第 5 步：完成开发

所有任务完成并验证后：
- 宣布："我正在使用完成开发分支技能来完成这项工作。"
- **必需子技能：** 使用 superpowers:finishing-a-development-branch
- 按该技能验证测试、展示选项、执行选择

## 何时停下求助

**立即停止执行当：**
- 在批次中间遇到阻碍（缺少依赖、测试失败、指令不清）
- 计划有导致无法开始的关键缺口
- 你不理解某个指令
- 验证反复失败

**请求澄清而不是猜测。**

## 何时回到前面的步骤

**在以下情况返回审查（第 1 步）：**
- 搭档根据你的反馈更新了计划
- 基本方法需要重新思考

**不要硬闯阻碍** — 停下来问。

## 注意事项
- 首先批判性审查计划
- 严格按计划步骤执行
- 不要跳过验证
- 计划中提到技能时引用对应技能
- 批次之间：只报告并等待
- 遇到阻碍时停下，不要猜测
- 未经用户明确同意，绝不在 main/master 分支上开始实现

## 集成

**必需的工作流技能：**
- **using-git-worktrees** - 必需：开始前设置隔离工作空间
- **writing-plans** - 创建此技能执行的计划
- **finishing-a-development-branch** - 所有任务完成后完成开发
