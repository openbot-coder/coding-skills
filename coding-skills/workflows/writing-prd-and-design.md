# 编写 PRD 和详细设计

## 适用范围

| 项目规模 | 功能点 | 文档要求 |
|----------|--------|----------|
| **脚本类** | ≤ 50 FP | 无需文档，使用 `script-development` |
| **中小型** | < 1000 FP | 1 份 PRD + 1 份设计文档 |
| **大型** | ≥ 1000 FP | 按模块拆分，每个模块独立 PRD + 设计 |

> **大型项目提示**：功能点 ≥ 1000 时，将系统拆分为多个模块，每个模块独立执行完整的 PRD → 设计 → 实施流程。

## 概述

在头脑风暴获得批准后，将需求转化为 PRD 文档和详细设计文档。

- **PRD** — 回答"做什么"（功能、用户价值、验收标准）
- **详细设计** — 回答"怎么做"（架构、技术方案、组件设计）

## 流程

```
brainstorming → writing-prd-and-design → writing-plans → 执行
```

## 检查清单

完成以下任务：

1. **编写 PRD 文档** — 包含产品概述、目标用户、功能需求、验收标准
2. **评审 PRD** — 与相关方确认需求完整性
3. **编写详细设计** — 包含架构、模块、接口、数据模型
4. **评审设计** — 与开发团队确认技术可行性
5. **保存文档** — 提交到 `docs/plans/` 目录
6. **过渡到实现** — 调用 writing-plans 技能创建实现计划

## 关键原则

- **PRD 关注业务价值** — 回答"为什么"和"做什么"
- **设计关注技术实现** — 回答"怎么做"
- **评审是必需的** — 文档需要相关方批准
- **文档先行** — 在触碰代码之前完成文档
